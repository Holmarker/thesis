Self-contained runbook for finishing the historical seasons on another device.
Written 2026-07-10. General background in `SCRAPING_RUNBOOK.md`; this file is
everything you need for these two seasons specifically.

## Setup on the device (once)

1. `git clone https://github.com/Holmarker/thesis.git && cd thesis`
   (or `git pull` if already cloned).
2. R 4.x with: `dplyr`, `jsonlite`, `lubridate`, `readr`, `stringr`, `tibble`.
3. Copy from OneDrive/main machine (data/ is gitignored):
   - `data/historical/fotmob_cookie.txt` (auth — required).
   - The existing checkpoint files for these two seasons (so resume-skip
     works and you only fetch what's missing).
4. `mkdir -p data/historical/checkpoints/ratings data/historical/match_fixtures data/historical/logs`

## Configs

- **2023/24**: exists — `config/league_config_resolved_2023_2024_clean.csv`.
- **2022/23**: does not exist yet. Create it from the 23/24 config by
  shifting the season dates one year back:

  ```bash
  sed 's/2023-07-01/2022-07-01/; s/2024-05-31/2023-05-31/' \
    config/league_config_resolved_2023_2024_clean.csv \
    > config/league_config_resolved_2022_2023.csv
  ```

  Note: calendar-year leagues (Argentina, Brazil A/B, Norway, Sweden) run
  Jan–Dec; the scraper handles that via fixture dates, but sanity-check the
  first calendar-year league's row counts before letting it run through.

## The command

Always from the repo root. The scraper is fully resumable: it re-fetches
fixture lists but skips every match ID already in the checkpoint file, so
re-running after any interruption is safe and cheap.

```bash
# 2023/24 — remaining leagues (safe to pass the FULL config;
# the 15 finished leagues are skipped via match-ID dedup)
FOTMOB_COOKIE_FILE=data/historical/fotmob_cookie.txt FOTMOB_REQUEST_DELAY=1 \
Rscript scripts/01_scraping/scrape_historical_fotmob_match_ratings.R \
  "2023/2024" config/league_config_resolved_2023_2024_clean.csv

# 2022/23 — everything except the top-5 (also safe with the full config)
FOTMOB_COOKIE_FILE=data/historical/fotmob_cookie.txt FOTMOB_REQUEST_DELAY=1 \
Rscript scripts/01_scraping/scrape_historical_fotmob_match_ratings.R \
  "2022/2023" config/league_config_resolved_2022_2023.csv
```

Do 2023/24 first (21 leagues) then 2022/23 (31 leagues). Rough sizing: a
full league-season is ~300–600 matches ≈ 300–600 requests, and FotMob allows
~600–4,000 requests per window — expect a handful of rate-limit pauses per
season.

## Rate limiting — no fixed cooldowns

FotMob blocks by silently returning **empty** responses (log shows leagues
producing 0 rows / `Saved fixtures for X: 0`); blocks clear after ~45–90 min.
Don't sleep on a timer between cycles — retry immediately and only wait when
the limiter is actually firing. This loop does that by checking whether the
last cycle added any data:

```bash
#!/bin/bash
# run as:  caffeinate -is bash scrape_loop.sh "2023/2024" config/league_config_resolved_2023_2024_clean.csv
SEASON="$1"; CONFIG="$2"; DIR=data/historical/checkpoints/ratings
while true; do
  before=$(cat $DIR/*_ratings.csv 2>/dev/null | wc -l)
  FOTMOB_COOKIE_FILE=data/historical/fotmob_cookie.txt FOTMOB_REQUEST_DELAY=1 \
    Rscript scripts/01_scraping/scrape_historical_fotmob_match_ratings.R \
    "$SEASON" "$CONFIG"
  after=$(cat $DIR/*_ratings.csv 2>/dev/null | wc -l)
  if [ "$after" -le "$before" ]; then
    echo "$(date): no new rows -> rate limited, waiting 30 min"; sleep 1800
  else
    echo "$(date): +$((after-before)) rows, continuing immediately"
  fi
done
```

Stop it with Ctrl-C when a full pass adds nothing AND every league's
checkpoint has real content (then it's genuinely done, not blocked).
`caffeinate -is` keeps the Mac awake; closing the lid/session kills the loop.

## When done: getting it into the pipeline (main machine)

Copy/sync the checkpoint files back into
`data/historical/checkpoints/ratings/` on the main machine, then:

```bash
# integrate each season into the master files
FOTMOB_HISTORICAL_SEASON_SLUG=2023-2024 Rscript scripts/03_build/integrate_historical_fotmob_ratings.R
FOTMOB_HISTORICAL_SEASON_SLUG=2022-2023 Rscript scripts/03_build/integrate_historical_fotmob_ratings.R

# rebuild panels (full build incl. backward fill, then augment)
Rscript scripts/03_build/build_fotmob_analysis_panel.R      # needs OTP db synced
Rscript scripts/03_build/augment_fotmob_panels_from_existing.R

# then rerun the analysis suite + tables (or just ask Claude to do this part)
```
